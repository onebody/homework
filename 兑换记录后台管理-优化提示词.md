# 兑换记录后台管理系统 — 优化提示词

## 项目背景

基于现有暑假作业打卡系统（`summer-homework-checkin/`），需要完善**兑换记录后台管理**功能，实现兑换记录列表展示、管理员审核兑现/拒绝、状态追踪、移动端适配。

---

## 一、业务规则

### 1.1 奖品分类与兑现方式

| 奖品类型 | `is_lottery_ticket` | 获取方式 | 兑现方式 |
|----------|---------------------|---------|---------|
| 抽奖机会（虚拟） | `True` | 积分兑换 | **系统自动兑现**，直接发放抽奖券 |
| 实物奖品（文具/户外/兴趣） | `False` | 积分兑换 | **待管理员手动核实兑现** |
| 实物奖品（文具/户外/兴趣） | `False` | 抽奖获得 | **待管理员手动核实兑现** |

### 1.2 兑换记录状态流转

```
积分兑换实物奖品 → pending（待发放）
                    ↓ 管理员操作
                fulfilled（已兑现）或 rejected（已拒绝，退还积分）

积分兑换抽奖机会 → fulfilled（系统自动兑现，无需管理员操作）

抽奖获得实物奖品 → pending（待发放）
                    ↓ 管理员操作
                fulfilled（已兑现）或 rejected（已拒绝）
```

`Redemption` 表已包含审核字段：

```python
class Redemption(Base):
    __tablename__ = "redemptions"
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False, index=True)
    prize_id = Column(Integer, ForeignKey("prizes.id"), nullable=False)
    prize_name = Column(String(128), nullable=False)
    cost_points = Column(Integer, default=0)
    redeemed_at = Column(DateTime, default=datetime.utcnow)
    status = Column(String(16), default="pending")  # pending|fulfilled|replaced|rejected|cancelled
    replaced_by = Column(Integer, ForeignKey("redemptions.id"), nullable=True)
    note = Column(String(256), nullable=True)
    review_note = Column(String(256), nullable=True)       # 审核备注
    reviewed_by = Column(Integer, ForeignKey("users.id"), nullable=True)  # 操作管理员ID
    reviewed_at = Column(DateTime, nullable=True)           # 操作时间
```

## 七、安全性要求

1. **JWT 鉴权**：所有管理接口需 `Authorization: Bearer <token>`
2. **角色校验**：`require_role("admin")`，非管理员返回 403
3. **防重复操作**：`status != "pending"` 时拒绝操作，返回 400
4. **操作审计**：记录 `reviewed_by`、`reviewed_at`、`review_note`
5. **拒绝时退还积分**：`user.points += r.cost_points`
