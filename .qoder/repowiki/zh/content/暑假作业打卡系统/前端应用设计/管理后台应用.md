## 引言
本设计文档面向"暑假作业打卡系统"的管理后台，聚焦于基于 Vue 3 的单页管理界面与企业级功能实现。内容覆盖管理员认证、用户管理、打卡审核、奖品管理、兑换记录审核、闯关任务管理与数据统计等模块；并深入阐述权限控制机制、数据表格操作、批量处理、报表生成、复杂表单、图片预览与上传、实时状态更新等企业级特性。同时给出与后端 FastAPI 管理 API 的交互模式、筛选与搜索策略，以及最佳实践与性能优化建议，帮助开发者快速构建高效稳定的管理后台。

**更新** 管理后台前端已完全重构，采用现代化的响应式卡片布局设计，集成Chart.js实现交互式数据可视化功能。新增30天打卡趋势分析、用户分布图表等数据展示功能，大幅提升用户体验和数据洞察能力。

## 项目结构
本项目采用前后端分离架构：
- 后端：FastAPI + SQLAlchemy + SQLite，提供 RESTful API，静态托管学生端与管理端页面。
- 前端（管理端）：Vue 3 CDN 单页应用，独立部署在 /admin 路径下，包含登录、概览、奖品、用户、打卡审核、兑换审核、闯关任务管理、密码修改等页面。

## 架构总览
管理后台整体由"前端管理 SPA + 后端 FastAPI 服务 + 数据库"构成。前端通过统一 API 方法携带 Bearer Token 访问受保护接口；后端通过依赖注入解析令牌、校验角色，再调用服务层执行业务规则，最终持久化到数据库。

```mermaid
sequenceDiagram
participant Admin as "管理员浏览器"
participant FE as "管理前端 app.js (Vue 3)"
participant Chart as "Chart.js 可视化"
participant API as "FastAPI 路由"
participant Auth as "安全/鉴权"
participant Svc as "服务层"
participant DB as "数据库"
Admin->>FE : 打开 /admin
FE->>API : POST /api/auth/login {username,password}
API->>Auth : 验证密码/签发Token
Auth-->>API : {access_token,user}
API-->>FE : 返回 token 与用户信息
FE->>FE : 保存 token 到 localStorage
Admin->>FE : 进入"修改密码"
FE->>API : PUT /api/auth/password {old_password,new_password}
API->>Auth : 验证原密码/校验新密码长度
Auth-->>API : 密码验证结果
API->>DB : 更新密码哈希
DB-->>API : 确认更新
API-->>FE : 返回成功消息
FE->>Chart : 渲染数据可视化图表
Chart-->>Admin : 显示交互式图表
```

## 详细组件分析

### 管理员认证与权限控制
- 登录流程
  - 前端提交用户名/密码至 /api/auth/login，后端校验成功后返回 access_token 与用户信息。
  - 前端将 token 写入 localStorage，并在后续请求头中附加 Authorization: Bearer <token>。
- 权限控制
  - 使用 HTTPBearer 自动解析令牌，decode_token 校验签名与过期时间。
  - require_role("admin") 确保仅管理员可访问管理接口。
- 错误处理
  - 未提供或无效令牌返回 401；非管理员返回 403；前端统一捕获并提示。

```mermaid
flowchart TD

Login --> Verify{"用户名/密码正确?"}
Verify -- 否 --> Err["返回 401 错误"]
Verify -- 是 --> Issue["签发 Token 并返回用户信息"]
Issue --> Save["前端保存 token 到 localStorage"]
Save --> Next["后续请求携带 Authorization"]
Next --> End(["结束"])
```

### 管理员密码修改功能
- **新增功能概述**
  - 在侧边栏导航中新增"🔑 修改密码"菜单项，提供完整的前端界面和后端API支持。
  - 实现与客户端一致的密码修改逻辑和安全验证规则。
- **前端实现**
  - 独立的密码修改页面，包含原密码输入、新密码输入、确认新密码输入字段。
  - 实时表单验证：原密码必填、新密码至少4位、两次密码必须一致、新密码不能与原密码相同。
  - 加载状态管理，防止重复提交。
  - 成功/失败的用户反馈提示。
- **后端实现**
  - PUT /api/auth/password 接口，需要管理员身份认证。
  - 验证原密码正确性，检查新密码长度要求。
  - 使用PBKDF2算法重新计算密码哈希和盐值。
  - 事务性更新用户密码信息。
- **安全特性**
  - 前后端双重验证确保密码修改的安全性。
  - 支持空盐值兼容旧数据格式。
  - 统一的错误处理和用户友好提示。

## 故障排查指南
- 认证失败
  - 检查 Authorization 头是否正确携带；确认 Token 未过期。
- 权限不足
  - 确认当前用户 role 为 admin；后端 require_role 校验失败返回 403。
- **密码修改失败**
  - 检查原密码是否正确；确认新密码长度至少4位；确保两次密码输入一致。
  - 验证新密码不能与原密码相同；检查网络连接和服务器状态。
- 图片上传失败
  - 检查文件格式与大小限制；确认 /api/checkin/upload 返回 photo_url。
- 审核报错
  - 确认记录状态为 pending；注意重复操作与备注必填项。
- 挑战任务问题
  - 检查任务状态是否为active或scheduled且unlock_at时间已过。
  - 确认学生端有权限访问任务（任务已解锁且未被完成）。
- **静态资源加载问题**
  - 检查CSS和JS文件路径是否为相对路径 `./admin.css` 和 `./app.js`。
  - 确认Vue.js CDN地址 `https://cdn.bootcdn.net/ajax/libs/vue/3.4.27/vue.global.prod.js` 可访问。
  - 验证子路径部署时资源路径是否正确解析。
- **Chart.js 相关问题**
  - 检查Chart.js库是否正确加载和初始化。
  - 确认图表容器元素存在且具有正确的尺寸。
  - 验证数据格式是否符合Chart.js要求。
  - 检查移动端触摸事件是否正常触发。
