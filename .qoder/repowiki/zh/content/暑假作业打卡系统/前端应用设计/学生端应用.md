## 引言
本设计文档面向"暑假作业打卡系统"的学生端（H5，基于 Vue 3 CDN），围绕用户认证、打卡流程、人脸识别采集、积分商城与抽奖系统、双角色（学生/家长）机制、状态管理与组件通信、文件上传、地理位置获取、表单验证、响应式布局、错误处理策略以及 API 封装与数据同步等主题进行系统化阐述。目标是帮助开发者快速理解并扩展学生端功能。

**更新** 本次更新重点完成了学生端应用导航系统的重构，将原有的四菜单项重新组织为五个主要功能模块（首页、打卡与挑战、抽奖、商城、我的），并将打卡和挑战功能整合到统一的页面中，提升了用户体验和功能集中度。同时增强了前端基础路径支持和静态资源URL处理能力。

## 项目结构
学生端采用前后端分离架构：
- 前端：Vue 3 单页应用，通过 bootcdn.net CDN 引入，免构建；静态资源由后端托管。
- 后端：FastAPI 提供 RESTful API，统一挂载路由、静态资源与 CORS。

- 认证与会话
  - 注册/登录返回访问令牌，前端持久化到本地存储并在后续请求中携带 Authorization 头。

  - 支持多孩子管理，可切换当前操作主体，自动刷新相关数据。
  - 绑定成功后自动导航到首页或保持当前视图。
  - 增强了无孩子场景的数据初始化，避免家长登录时出现空白页面。
- **更新** 打卡与挑战整合功能
  - 将打卡和挑战功能整合到统一页面，提供集中的任务管理和进度跟踪。

- 人脸识别采集

  - 增强了streak对象结构，包含完整的统计信息字段。
- 前端状态与通信
  - 单一 Vue 实例集中管理状态；统一的 api 方法封装 fetch，统一鉴权与错误提示；根据 isParent 分支选择不同 API。
  - 增强了数据结构的健壮性，提供更好的向后兼容性。
  - **新增** 静态资源路径优化，支持子路径部署和CDN加速。

#### 前端验证机制
- **必填项验证**：检查原密码是否为空
- **长度验证**：新密码至少需要 4 个字符
- **一致性验证**：新密码与确认密码必须一致
- **安全性验证**：新密码不能与原密码相同
- **防重复提交**：通过 pwdBusy 标志位防止重复提交请求
- **用户体验**：修改成功后自动清空表单并关闭编辑状态

#### API 通信处理
- **请求封装**：使用统一的 api 方法处理 HTTP 请求
- **错误处理**：捕获网络错误和业务错误，显示友好提示信息
- **状态管理**：通过 showPwdForm 控制表单显示状态
- **用户反馈**：通过 showToast 方法提供操作结果反馈

#### 后端密码修改接口
- **权限校验**：需要有效的用户认证令牌
- **原密码验证**：验证用户提供的原密码是否正确
- **新密码验证**：检查新密码长度要求（至少4位）
- **安全更新**：使用 PBKDF2 算法对新密码进行哈希处理
- **事务处理**：确保密码更新的原子性和一致性

#### 数据结构定义

## 故障排查指南
- 401 未授权
  - 检查前端是否携带 Authorization 头；确认 Token 未过期。
- 人脸比对失败
  - 确认已采集人脸底图；检查模型可用性；必要时调整阈值或切换策略。
- 补卡失败
  - 检查目标日期是否在暑假范围内、是否超过月度次数上限、是否已有有效打卡。
- 兑换失败
  - 检查积分是否足够、奖品是否上架、库存是否为 0。
- 定位失败
  - 浏览器可能拒绝定位；仍允许提交但会标记风险。
- 绑定失败
  - 检查孩子用户名是否正确、绑定码是否匹配、是否已重复绑定。
- 密码修改失败
  - 检查原密码是否正确、新密码长度是否满足要求、两次密码输入是否一致。
  - 确认网络连接正常，后端密码修改API可正常访问。
- 家长登录空白页面
  - 确认loadChildren()函数中的无孩子场景数据初始化逻辑是否正常执行。
  - 检查streak、today、mall对象的所有必需字段是否都已正确初始化。
- **新增** 导航系统问题
  - 检查五菜单项导航是否正确加载和渲染。
  - 确认导航路由配置是否正确，页面切换是否正常。
  - 验证打卡与挑战整合页面的数据加载和状态同步。
- **新增** 静态资源加载失败
  - 检查bootcdn.net CDN连接是否正常，确认Vue.js库是否正确加载。
  - 验证CSS和JS文件的相对路径是否正确，特别是在子路径部署场景下。
  - 检查BASE_PATH自动检测逻辑，确认基础路径设置是否正确。
  - 使用浏览器开发者工具检查网络请求，确认静态资源URL拼接是否正确。
