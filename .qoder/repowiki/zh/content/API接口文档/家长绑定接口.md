## 简介
本文件面向"家长-孩子绑定关系"的API，覆盖以下能力：
- 家长账号注册、登录与鉴权
- 孩子账号注册与绑定码生成机制
- 家长绑定孩子、查询已绑定的孩子列表
- **新增**：家长解绑孩子、管理绑定关系
- 家长查看孩子学习进度（连续打卡、今日状态等）
- 接收打卡通知（站内通知）
- 家长代孩子打卡、兑换、抽奖等关联操作
- 权限控制与安全性保障（角色校验、绑定关系校验）
- 数据模型与状态管理（绑定关系表、通知表、用户统计字段）
- 错误处理方案与用户体验优化建议

该功能基于 FastAPI + SQLAlchemy 实现，采用前后端分离架构。学生端 H5 提供绑定码展示；家长端通过绑定码完成绑定并接收实时通知。**最新更新**：新增了完整的家长解绑功能，包括后端DELETE /api/parent/unbind/{student_id}接口和前端解绑界面，增强了家长对孩子绑定关系的管理能力。

## 项目结构
与"家长绑定"相关的关键路径与职责：
- 路由层
  - /api/auth：注册、登录、获取当前用户信息

- 服务层
  - checkin_service：打卡业务逻辑（含通知触发）
  - notify_service：站内通知写入
- 数据模型
  - User：统一用户表（student/parent/admin），包含绑定码、统计冗余字段
  - StudentParent：家长-孩子多对多绑定关系
  - Notification：站内通知记录
- 请求/响应模式
  - schemas：BindRequest、ChildSummary、NotificationOut 等
- **新增**：数据迁移工具
  - migrate.py：数据库迁移和数据回填功能

## 核心组件
- 认证与令牌
  - 注册：支持 student/parent 两种角色；学生注册后自动生成绑定码
  - 登录：返回访问令牌与用户基本信息
  - 当前用户：通过依赖注入获取当前登录用户上下文
- 家长绑定管理
  - 绑定：仅家长可绑定；根据孩子用户名+绑定码校验；重复绑定幂等
  - **新增**：解绑：仅家长可解绑；校验绑定关系存在性；删除绑定关系
  - 查询：列出家长已绑定的所有孩子及其关键状态
- 通知
  - 家长通知：按 recipient_role=parent 过滤
  - 打卡通知：提交打卡时自动向孩子与家长发送站内通知
- 数据模型
  - User.bind_code：学生绑定码（Sxxxxx 格式）
  - StudentParent：绑定关系（多对多）
  - Notification：通知记录（type、title、content、read 等）
- **新增**：数据迁移增强
  - backfill_bind_codes：为已有学生补全缺失的绑定码

## 架构总览
家长绑定链路涉及"认证 → 绑定/解绑 → 通知 → 查询"的闭环。下图展示了从家长注册到绑定成功、再到收到通知的核心交互。

```mermaid
sequenceDiagram
participant Client as "客户端"
participant Auth as "/api/auth"
participant Parent as "/api/parent"
participant DB as "数据库"
participant Notify as "notify_service"
participant Migrate as "migrate.py"
Client->>Auth : POST /api/auth/register(student/parent)
Auth->>DB : 创建用户(学生则生成绑定码)
Auth-->>Client : 返回 access_token + user
Client->>Parent : POST /api/parent/bind(child_username, bind_code)
Parent->>DB : 校验家长角色/绑定码/重复绑定
Parent->>DB : 写入 StudentParent 绑定关系
Parent-->>Client : 绑定成功
Client->>Parent : DELETE /api/parent/unbind/{student_id}
Parent->>DB : 校验家长角色/绑定关系存在
Parent->>DB : 删除 StudentParent 绑定关系
Parent-->>Client : 解绑成功
Client->>Parent : GET /api/parent/notifications
Parent->>DB : 查询 parent 的通知
Parent-->>Client : 返回通知列表
Note over Client,Migrate : 系统启动时执行 backfill_bind_codes 补全历史数据
Note over Client,Notify : 当学生提交打卡时，系统会调用 notify_parents_of_student 推送通知给家长
```

### 家长绑定孩子
- 接口：POST /api/parent/bind
- 入参：child_username、bind_code
- 校验逻辑：
  - 仅家长角色可绑定
  - 孩子存在且为 student 角色
  - 绑定码匹配
  - 重复绑定幂等返回"已绑定"
- 结果：写入 StudentParent 绑定关系，返回绑定成功
- **前端实现**：家长登录后显示绑定界面，支持输入孩子用户名和绑定码进行绑定

```mermaid
flowchart TD
Start(["开始"]) --> CheckRole["校验家长角色"]
CheckRole --> |非家长| Err403["返回 403 无权限"]
CheckRole --> |家长| FindChild["按 child_username 查找学生"]
FindChild --> ChildFound{"找到学生?"}
ChildFound --> |否| Err400a["返回 400 孩子不存在"]

```

## 故障排查指南
- 绑定失败
  - 403 无权限：确认当前用户角色为 parent
  - 400 孩子不存在/绑定码错误：核对 child_username 与 bind_code
  - 重复绑定：幂等返回"已绑定"，无需重试
- **新增**：解绑失败
  - 403 无权限：确认当前用户角色为 parent
  - 已解绑：如果返回"该孩子已解绑"，说明绑定关系不存在，无需重复操作
  - 网络错误：检查网络连接和服务器状态
- 通知未收到
  - 确认学生已提交打卡且系统触发了 notify_parents_of_student
  - 检查 Notification 表中 recipient_role=parent 的记录是否存在
- 家长代打卡失败
  - 照片尺寸/格式不合规
  - 人脸策略拒绝（已采集底图且比对不通过）
  - 地理位置异常标记风险（不影响提交，但后台可见）
- **新增**：前端解绑问题
  - 确认解绑按钮是否正确绑定到 unbindChild 方法
  - 检查 API 调用是否正确传递 student_id 参数
  - 验证解绑成功后是否正确刷新孩子列表
