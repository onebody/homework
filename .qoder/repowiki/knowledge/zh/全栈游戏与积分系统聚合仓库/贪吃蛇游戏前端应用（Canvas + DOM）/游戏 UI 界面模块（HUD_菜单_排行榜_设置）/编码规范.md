- 每个 UI 模块以命名空间对象形式组织，统一提供 `init(gameEngine)`、`bindEvents()`、可选的 `loadSettings()` 以及 `show()/hide()` 生命周期方法。
- 通过 `document.getElementById(...).classList.add/remove('active')` 切换屏幕可见性，而非创建/销毁 DOM 节点。
- UI 层不直接读写 localStorage，而是委托 `this.gameEngine.getStorageData()` / `setTimeLimit()` / `toggleSound()` 等引擎 API，保持视图与数据持久化解耦。
- DOM 事件使用 `addEventListener` 绑定，并在初始化时一次性注册，不在渲染路径中重复绑定。
- 模块末尾统一包含 `if (typeof module !== 'undefined' && module.exports) { module.exports = X; }` 以兼容 CommonJS。
