模块按职责分为两层：
- `core/` 承载游戏运行时对象：`GameEngine` 作为唯一入口，持有 canvas、状态机（IDLE/READY/PLAYING/PAUSED/GAME_OVER）和固定帧率累加器驱动的 `gameLoop`，协调 `Snake`、`Food`、障碍物数组以及粒子/飘字/死亡闪烁等视觉特效；`Collision` 为纯函数式对象，提供 wall/self/obstacle/food 四类检测并聚合为 `checkAll` 返回 `{wall,self,obstacle}`。
- `utils/` 提供跨层共享能力：`EventBus` 通过单例 `globalEventBus` 暴露 on/off/once/emit/clear，所有 UI 层（音频、成就、排行榜）通过订阅 `game:*` / `audio:play` 等命名空间事件与引擎解耦；`Constants` 集中定义网格尺寸、方向向量、难度/模式/食物类型/皮肤配色/默认设置与 localStorage key；`Helpers` 暴露随机数、坐标比较、防抖节流、震动、URL 参数等无状态工具。
依赖方向严格单向：`core/*` → `utils/*`，`utils/*` 之间互不引用；各文件末尾以 `module.exports` 兼容 Node 环境，但运行时通过全局常量（如 `GRID_SIZE`、`DIFFICULTY`、`GAME_STATE`、`FOOD_TYPE`、`SKIN_COLORS`、`STORAGE_KEY`、`vibrate`、`randomInt`、`isPositionInArray`、`getRandomFoodType`、`debounce`、`globalEventBus`）隐式耦合，形成“全局常量 + 全局事件总线”的轻量架构。
