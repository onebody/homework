- 启动顺序：`index.html` → `js/main.js` 在 `DOMContentLoaded` 中创建唯一 `GameEngine('game-canvas')` 实例并挂到 `window._gameEngine`，随后依次调用 `AchievementManager.init()`、`Menu/HUD/Settings/Leaderboard.init(gameEngine)`、`AudioManager.init()`。
- 通信契约：所有跨模块交互统一走 `utils/EventBus.js` 的 `globalEventBus`（如 UI 层发布 `audio:play`，core 层消费；成就系统监听游戏状态变更），避免直接互相引用。
- 输入路由：`main.js` 集中处理键盘、触摸滑动与虚拟方向键，仅把标准化 `DIRECTION.*` 常量交给 `gameEngine.handleInput()`，UI 与渲染不感知输入细节。
- 渲染边界：`render/CanvasRenderer.js` 由 core 内部驱动，UI 屏幕只操作 DOM 类名切换（`.active`）与 HUD 文本更新，两者互不侵入。
- 持久化与主题：`data/StorageManager.js` 提供 localStorage 读写，`themes/` 目录配合 CSS 变量实现运行时换肤，被 Menu/Settings 共同消费。
