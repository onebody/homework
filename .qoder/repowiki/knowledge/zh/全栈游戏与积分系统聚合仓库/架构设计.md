本仓库采用「多子模块 + 顶层设计文档」的聚合式结构：
- `summer_homework_checkin`（FastAPI 后端）与 `points_system`（独立 FastAPI 服务）共同承载「打卡赚积分 → 积分兑换奖品/抽奖券 → 加权随机抽奖」业务，二者在运行时可独立部署，通过统一的积分模型约定交互；
- `snake_game` 为纯前端 Canvas 单页应用，与两个后端子系统无运行时耦合，仅作为独立静态站点交付；
- 顶层 `snake-game-prd.md` / `snake-game-architecture.md` / `snake-game-project-plan.md` / `optimized-prompt.md` / `兑换记录后台管理需求提示词.md` 作为跨子模块的单一事实来源，分别约束贪吃蛇前端与打卡/积分后端的 PRD、架构、WBS 里程碑与 Prompt 模板，使三个子模块在同一份需求与计划下并行演进。
- 子模块之间不存在共享代码或进程间通信，隔离边界由目录划分与各自入口文件（如 `js/main.js`、FastAPI app 实例）保证。
