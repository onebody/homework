- 所有涉及余额/库存变动的操作都在同一个 SQLAlchemy Session 事务内完成，先做全部前置校验，通过后在同一事务中扣减并写入流水，统一 `db.commit()`，异常时 `db.rollback()` 并抛出 HTTPException。
- 账户权限不存独立状态位，而是由 `point_accounts.lottery_tickets >= 1` 派生，确保「券数 ↔ 抽奖解锁」始终一致。
- 每笔收支都落对应流水表（`point_ledgers` / `lottery_ticket_ledgers`），记录 `tx_type`、`amount`、`balance_after`、`ref_type`、`ref_id`，用于端到端对账。
- 并发敏感路径（`do_convert` / `do_draw`）包裹 `threading.Lock`，将「读-改-写」整体串行化，避免 SQLite 下的丢失更新。
- 路由文件按领域拆分（checkin / points / prize / redeem / convert / lottery / users），每个文件仅暴露一个 `router` 供 `main.py` 统一 `include_router` 注册。
