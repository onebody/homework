后端采用 FastAPI 分层：`app/main.py` 通过 `lifespan` 在启动时调用 `init_db()` 建表，并依次 `include_router` 注册 `checkin / points / prize / redeem / users / convert / lottery` 七个路由；静态前端目录 `frontend/` 通过 `StaticFiles(directory=..., html=True)` 挂载到 `/`，API 统一以 `/api` 前缀暴露。

- 数据层：`database.py` 提供引擎/会话/Base，`models.py` 用 SQLAlchemy 2.0 声明式 ORM 定义 10 张表（users、point_accounts、point_ledgers、checkins、prizes、redemptions、conversions、lottery_ticket_ledgers、lottery_prizes、lottery_draws），其中 `checkins` 通过 `UniqueConstraint(user_id, check_date)` 从数据库层兜底每日一次打卡。
- 业务层：`services/points_service.py` 封装打卡/兑换奖品的带事务逻辑；`services/lottery_service.py` 封装「积分换券 + 抽奖」，两者均遵循「先校验、再同事务内扣减+写流水、统一 commit/rollback」模式，并在 SQLite 演示场景下使用进程内 `threading.Lock` 串行化读-改-写，避免丢失更新。
- 配置集中：`config.py` 暴露 `POINTS_PER_CHECKIN / STREAK_BONUS_EVERY / POINTS_PER_TICKET / TICKETS_PER_DRAW` 等规则常量，便于环境变量覆盖。
- 启动脚本：`run.py` 通过 uvicorn 启动 `app.main:app`；`seed.py` 预置用户、奖品与奖池演示数据。
- 依赖方向：routers → services → models/database/config，前端无构建产物，纯原生 HTML/CSS/JS 由后端静态托管。
