本仓库包含三个独立的前端应用，各自采用原生 CSS + CSS Variables 的轻量级样式方案，未引入任何 CSS 框架或构建工具链。各模块风格相对独立，但遵循一致的变量驱动设计思路。

## 1. 样式系统概览

- **技术栈**：纯 CSS + CSS Variables（`:root` 自定义属性），无 Sass/Less、无 Tailwind/Bootstrap、无 CSS-in-JS。
- **组织方式**：每个子项目一个独立的 `*.css` 文件，按页面角色拆分（如 `student.css` / `admin.css`）。
- **设计令牌**：通过 `:root` 集中声明颜色、圆角、阴影等主题变量，支持运行时切换（如护眼模式、大字体模式）。
- **响应式策略**：基于 `@media` 断点（560px / 768px / 640px）做移动端适配，部分使用 Grid/Flex 自适应布局。

## 2. 各模块样式要点

### points-system/frontend/styles.css
- 蓝色主色调（`--primary: #3b6ef5`），卡片化 UI，圆角 16px，柔和阴影。
- 转盘抽奖动画通过 `transform` + `cubic-bezier` 缓动实现，弹窗使用 `popIn` 关键帧。
- 网格卡片 `.cards { grid-template-columns: 1fr 1fr 1fr }`，在 ≤560px 时退化为单列。

### snake-game/css/main.css
- 绿色游戏主题（`--primary-color: #4CAF50`），提供「护眼模式」`[data-eye-care="true"]` 和「大字体模式」`[data-large-font="true"]` 两套运行时主题开关。
- 屏幕管理通过 `.screen` + `.active` 类切换，配合 `fadeIn` 过渡。
- 移动端虚拟方向键 `.d-pad` 使用 3×3 Grid 布局，≤768px 自动显示。
- 成就解锁通知 `.achievement-notification` 从顶部滑入，带弹性缓动。

### summer-homework-checkin/frontend/student/student.css
- 蓝白渐变登录页，卡片圆角 18px，底部固定 TabBar。
- 打卡上传区 `.uploader` 使用虚线边框 + 绝对定位透明 input 实现拖拽点击。
- 双角色（孩子切换）通过 `.child-switch` 标签组实现。
- 积分商城奖品卡片 `.mprize` 使用 2 列 Grid 展示。

### summer-homework-checkin/frontend/admin/admin.css
- 深色侧边栏 + 浅色主内容区的经典后台布局 `.layout`。
- 内置完整图片查看器：全屏遮罩 + 缩放平移 + 缩略图导航 + 批量上传面板。
- 审核状态 badge 系统（`.review-badge.approved/rejected/pending`）统一配色。
- 移动端将侧边栏折叠为底部 Tab，表格切换为卡片列表 `.redeem-cards`。

## 3. 通用约定

| 约定 | 说明 |
|------|------|
| 命名空间 | 全部使用 BEM 风格的短横线命名（如 `.btn-primary`、`.card`、`.modal-card`），无全局污染风险 |
| 交互反馈 | 统一的 `.toast` 轻提示组件，位于底部居中；按钮 hover 使用 `transform: translateY(-2px)` 微浮起 |
| 模态框 | `.modal` + `.modal-card` 双层结构，背景半透明遮罩，最大宽度 360px，移动端全宽底部滑出 |
| 无障碍 | 输入框最小高度 44px（移动端触控友好），禁用 `-webkit-tap-highlight-color` |
| 字体栈 | 优先系统字体 `-apple-system, BlinkMacFont, "Segoe UI"`，中文回退 `"PingFang SC", "Microsoft YaHei"` |

## 4. 开发者应遵循的规则

1. **新增样式先查 `:root` 变量**，不要硬编码颜色值；如需新主题色，先在根变量中声明。
2. **组件样式保持单一职责**，一个 CSS 文件对应一个页面/功能域，避免跨模块复用导致耦合。
3. **响应式优先移动设备**，所有 `max-width` 断点下需验证布局完整性。
4. **动画统一缓动曲线**，参考现有 `cubic-bezier(.2,.9,.3,1)` 或 `ease-out`，避免突兀过渡。
5. **不引入第三方 CSS 库**，保持零依赖、可直接用浏览器打开的开发体验。

## 5. 已知不一致之处

- 三个模块的主色调不同（points-system 偏蓝、snake-game 偏绿、summer-homework 偏蓝），缺乏跨项目的统一 Design Token。
- 圆角半径在各模块间不统一（10px / 12px / 16px / 18px），建议收敛到 2-3 个标准值。
- 未建立共享的基础样式层（reset/base），各项目自行定义 `* { box-sizing: border-box }`。
