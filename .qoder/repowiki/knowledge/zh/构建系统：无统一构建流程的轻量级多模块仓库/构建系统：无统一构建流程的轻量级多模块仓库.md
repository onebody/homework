本仓库为聚合型学习/作业项目，包含三个独立子模块（points-system、summer-homework-checkin、snake-game），**不存在统一的构建系统**。各模块各自以「最小可用」方式运行：

- **Python 后端（两个 FastAPI 应用）**：依赖通过 `requirements.txt` 声明，启动方式为直接 `uvicorn` 命令行或 `run.py` 入口脚本；数据库使用 SQLite 文件，模型在应用启动时自动建表，无需迁移工具。
- **前端静态页面**：纯 HTML/CSS/JS，无任何打包器（webpack/vite）或包管理器（npm/yarn/pipenv/poetry），浏览器直接打开即可。
- **贪吃蛇游戏**：单页 Canvas 应用，同样零构建步骤。

仓库中未发现 Makefile、Dockerfile、docker-compose、`.github/workflows`、CI 流水线、版本发布脚本等任何构建/部署工件。README 中的安装说明仅为 `pip install -r requirements.txt` + 手动运行 Python 脚本，属于开发机本地调试模式。因此本仓库不具备可复现的构建与制品管理能力。
