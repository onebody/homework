# 功能模块说明文档

> 暑假作业打卡系统（三年级）· 功能模块与业务规则

## 一、角色与权限

| 角色 | role | 主要能力 |
| --- | --- | --- |
| 学生 | `student` | 打卡、闯关、抽奖、积分兑换、查看报告、采集人脸底图 |
| 家长 | `parent` | 绑定/解绑孩子，代孩子打卡/兑换/抽奖，查看孩子报告与通知 |
| 管理员 | `admin` | 概览统计、用户管理、打卡审核、兑换审核、奖品与闯关任务管理 |

## 二、学生端功能模块

学生端为移动端 H5 单页应用，底部导航共 **5 个主菜单**（v1.0 起）：

| 图标 | 菜单 | 视图（view） | 说明 |
| --- | --- | --- | --- |
| 🏠 | 首页 | `home` | 连续打卡概览、今日打卡状态、快捷入口 |
| 📝 | 打卡闯关 | `checkin-challenge` | 每日打卡 + 闯关任务（页内子 tab 切换） |
| 🎰 | 抽奖 | `lottery` | 转盘抽奖、抽奖券数量、中奖记录 |
| 🛍️ | 商城 | `mall` | 积分兑换奖品、我的兑换、替换 |
| 👤 | 我的 | `me` | 账号信息、人脸底图、改密码、打卡记录、（家长）孩子管理 |

### 2.1 首页
- **连续打卡卡片**：当前连续天数、历史最长、累计有效次数、当前积分、距解锁下一次抽奖的剩余天数。
- **今日状态**：已完成 / 待审核 / 未打卡三态，未打卡时提供「去打卡」快捷按钮跳转打卡闯关页。
- **快捷入口**：积分商城、报告、我的。

### 2.2 打卡闯关（合并页）
页内子 tab 在「📸 每日打卡」与「🏆 闯关任务」间切换，数据在进入页面时预加载，切换不重复请求。

**每日打卡（3 步内完成）**
1. 上传作业现场照片（必填，强制真实照片）
2. 补卡选项（可选）：勾选后选择补卡日期、原因、上传补充凭证
3. 获取位置并提交（正常打卡 +10 积分 / 补卡 +5 积分）

**闯关任务**
- 任务列表：展示任务名、奖励积分、开放状态（已开放/定时开放/未开放）、个人完成状态。
- 任务详情弹窗：查看任务说明与奖励，未完成时可填写文字说明 + 上传完成照片提交打卡。
- 我的闯关打卡记录：展示提交历史与审核状态（已通过/未通过/审核中）。

### 2.3 抽奖（转盘）
- **抽奖券数量**：页面顶部醒目展示当前可用抽奖资格（🎫 + 数字）。
- **转盘 UI**：`conic-gradient` 绘制彩色分区，分区由商城奖品（非抽奖券）动态生成 + 一个「谢谢参与」，中心「抽」按钮。
- **抽奖动画**：点击后转盘旋转 5 圈并精准落在结果分区（4s 缓动），落定后弹出中奖结果。
- **中奖记录**：展示历史抽奖结果列表。
- 每次抽奖消耗 1 张抽奖券；无券时按钮禁用并提示。

### 2.4 积分商城
- **积分余额**：打卡累计所得。
- **奖品兑换**：文具/户外/兴趣分类展示，实物奖品扣库存，抽奖券类奖品可反复兑换（兑换后增加抽奖资格）。
- **我的兑换**：兑换记录 + 「替换」功能（在库存变化时替换为其他等值奖品）。

### 2.5 我的
- 人脸打卡验证：采集/撤销正脸底图（仅学生）。
- 账号信息：昵称、身份、绑定码（学生，供家长绑定）、积分、连续天数、抽奖券。
- 密码设置：旧密码 → 新密码修改。
- 打卡记录：历史打卡与审核状态。
- 家长专属：已绑定孩子列表（可解绑）、绑定更多孩子。
- 学习报告入口：查看 / 下载 HTML 报告。

## 三、家长双角色

家长注册后凭孩子的**用户名 + 绑定码**绑定，可绑定多个孩子；家长端顶部提供孩子切换器。家长的所有操作都以「当前选中孩子」为对象，通过 `/api/parent/*` 端点在校验绑定关系后代为执行：

- 代孩子打卡（含补卡）
- 代孩子积分兑换 / 替换
- 代孩子抽奖
- 查看孩子的连续天数、积分、报告
- 接收孩子打卡/中奖的站内通知
- 解绑孩子（解绑后可重新绑定其他孩子）

## 四、防代打卡机制

打卡提交时由 `verification_service` 执行四重校验，结果综合为 `scene_check` 与风险等级并持久化，供后台审核：

1. **照片真实性**（`utils/image.py`）：校验为合法 JPEG/PNG、体积 ≥ 5KB、最小边长 ≥ 200px，过滤占位图/缩略图。
2. **地理位置一致性**（`utils/geo.py`）：现场经纬度与账号常用位置的 Haversine 距离超过 `GEO_THRESHOLD_METERS`（默认 1500m）时置 `geo_flag`，后台高亮。
3. **人脸 1:1 本人比对**（`face_service.py`，核心）：现场照与已采集底图经 insightface（buffalo_l）提取 512 维特征做余弦相似度，低于 `FACE_MATCH_THRESHOLD`（默认 0.4）时——
   - `FACE_MODE_ON_ENROLLED=enforce`（默认）：**拒绝打卡**；
   - `soft`：仅标记高风险仍记录。
4. **场景合规综合判定**：汇总以上为 `scene_check`（pass/warn/pending），记录 `face_status / face_score / face_flag`。

> **降级安全模式**：无外网无法下载模型时自动降级——已采集底图的账号打卡被拒绝（防绕过），未采集账号正常。采集/比对/拦截链路完整可演示。数据模型已预留 `face_embedding` 字段，可平滑扩展 1:N 全班人脸库检索。

## 五、打卡与激励规则

| 规则 | 值 | 配置项 |
| --- | --- | --- |
| 每日有效打卡 | 同一自然日仅 1 条正常打卡 | — |
| 正常打卡积分 | +10 | `CHECKIN_POINTS` |
| 补卡积分 | +5 | `MAKEUP_POINTS` |
| 每月补卡上限 | 3 次 | `MAX_MAKEUP_PER_MONTH` |
| 抽奖解锁 | 连续有效打卡每满 7 天 +1 资格 | `LOTTERY_STREAK_THRESHOLD` |
| 地理阈值 | 1500 米 | `GEO_THRESHOLD_METERS` |
| 人脸阈值 | 0.4（余弦） | `FACE_MATCH_THRESHOLD` |

- **连续天数**：`recompute_and_grant` 基于有效打卡日期计算 `current_streak / longest_streak`。
- **抽奖资格**：永久累积、不可折现/转让；`last_7_milestone` 记录已解锁的里程碑避免重复发放。

## 六、后台管理模块

后台管理页（`/admin/`）供管理员（老师）使用：

- **概览统计**（`/api/admin/stats`）：用户数、打卡数、待审核数等。
- **用户管理**（`/api/admin/users`）：查看全部用户。
- **打卡审核**（`/api/admin/checkins`）：列表（含位置/人脸异常高亮）、待审核计数、逐条审核（通过/拒绝 + 备注）。审核通过后打卡计入有效、发放积分与抽奖资格。
- **兑换审核**（`/api/admin/redemptions`）：兑换记录列表、详情、审核（发放/驳回）。
- **奖品管理**（`/api/admin/prizes`）：预设 12 项 + 自定义，增删改、概率/库存/上下架、抽奖券配置。
- **闯关任务管理**（`/api/challenge/admin/tasks`）：任务 CRUD、定时/手动开放、闯关打卡审核。

---

## 七、报表模块

- **学生报告**（`/api/report/me` JSON、`/api/report/me/html` 可视化）：暑假全周期（默认 2026-07-01 ~ 08-31）的打卡频率、最长连续、完成情况，HTML 版可打印/下载。
- **家长查看孩子报告**（`/api/parent/child-report/{id}[/html]`）。

## 八、配套系统：打卡积分兑换系统（points-system）

独立部署于端口 8001 的轻量积分兑换系统，提供奖品（prizes）浏览与兑换能力，与主系统并列运行、数据独立（`points-data` 卷）。其前端为 Vue3 CDN 单页，后端同为 FastAPI + SQLite。
